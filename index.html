1)OWASP:

  • OWASP stands for Open Web Application Security Project
  • OWASP is a non-profit global organization that focuses on improving Web application security
  • Roles:
    1. Awareness and Education:
      • OWASP educates the people on Web Application security through seminars, workshops, etc...
    2. OWASP Top 10:
      • OWASP publishes the OWASP Top 10
      • It consists of the top web application security vulnerabilities
      • It fixes the security vulnerabilities like Injection, Cross-Site Scripting(XSS), Broken Authentication
    3. Tools:
      • OWASP provides tools like:
        1. OWASP ZAP - Used for testing
        2. Dependency-Check - Used for identifying vulnerabilities
      • These tools are used to improve the web application security
    4. Frameworks:
      • OWASP provides frameworks like:
        1. OWASP ASVS
        2. SAMM
      • These frameworks are used to improve the web application security

2)SQL Injection:

  • In SQL Injection, the Attacker Injects harmful commands in SQL Queries for accessing the sensitive Information
  • Injection Risks:
    1. Data breach:
      • Attackers can read, modify, or delete the data from databases
    2. Integrity breach:
      • Injecting Harmful commands in SQL Queries can change the records in Databases
    3. Availability impact:
      • Injecting Harmful commands in SQL Queries can degrade the Database
    4. Authentication breach:
      • Attackers can easily log in without valid credentials
    5. reputation damage:
      • Injecting Harmful commands in SQL Queries can damage the organization reputation
  • Mitigation techniques:
    1. Use parameterized queries:
      • user inputs are treated as data, not executable code
      • It prevents the attackers from accessing the sensitive information
      • Example:
        • a = prepare("select * from user where name = ?");
        • a.bind();
        • a.execute();
    2. Use Stored Procedure:
      • Execute SQL Queries by using a stored procedure with parameters
    3. WAF:
      • WAF stands for Web Application Firewalls
      • WAF is used to detect and block the injection patterns
    4. Verification:
      • Always verify the user input before using it
    5. Input Validation:
      • Always validate the user input to prevent security attacks
    6. Error Handling:
      • Hide the Error Messages from users
      • watch error messages from the server for analysis

3)Cross-Site Scripting(XSS):

  • In Cross-Site Scripting(XSS), the Attacker injects harmful scripts(java Script) into the webpages for accessing the sensitive Information
  • Types of XSS attacks:
    • Stored XSS
    • Reflected XSS
    • DOM-based XSS
  • Example:
    • <input name="a">
    • <script> alert("Hacked"); </script>
  • How does it work?
    1. finding Injection Point:
      • The attacker finds the place where the input is not checked
    2. Create Script:
      • Create a tiny script to steal the data
    3. Running Code:
      • This Harmful code directly runs in the victim's browsers without any restrictions
    4. Tricking users:
      • The data is stored in stored XSS and tricking the users into clicking the malicious link in Reflected XSS
    5. steals the data:
      • Finally, the attacker steals the data, sensitive information and credentials from the users
  • Mitigation techniques:
    1. use CSP:
      • CSP stands for Content Security Policy
      • It restricts the inline and external script sources
    2. use HTTPOnly:
      • Use HTTPOnly cookies to keep the data safe from JavaScript
    3. Avoid Unsafe functions:
      • Avoid unsafe functions like eval(), innerHTML, etc...

4)Broken Authentication:

  • Broken Authentication happens when there is a problem in verifying the users
  • Broken session management happens when there is a problem in managing the sessions
  • common vulnerabilities:
    1. weak passwords:
      • If the user uses weak passwords instead of strong ones makes the attackers easily access the login credentials via brute force
    2. Resetting Passwords:
      • Resetting Passwords is unsafe when tokens are predictable
    3. Revealing Username:
      • Error Messages that reveal the usernames make the attackers easily access the login credentials
    4. Session Fixation:
      • Attacker fixes the session ID for the users and uses it after the users log in
  • Mitigation Techniques:
    1. Strong Passwords:
      • Use a strong Password instead of weak ones for strong authentication
    2. Protect Password Reset:
      • Protect Password Reset with one-time tokens
    3. prevent session fixation:
      • Generate a new session ID after authentication to prevent session fixation
    4. Secure Cookies:
      • Use Secure Cookies like HTTPS, HTTPOnly to prevent security attacks

5)Broken Access Control:

  • Broken access control happens when the data is exposed and can be changed by the attackers
  • Broken access control is dangerous because it allows unauthorized users to access and manipulate the data due to weak access control
  • Example:
    • IDOR:
      • IDOR stands for Insecure Direct Object Reference
      • Internal IDs like file names, DB Keys, URLs are exposed and can be changed by the attackers
    • How does it work?
      • The app exposes the Internal ID in the URL or form parameter
      • Example:
        • Invoice?id=1234
      • Attacker changes the ID to access People's data
      • Example:
        • id=1234 to id=4321
      • If servers do not recheck the authorization, it allows attackers to access the People's data

6)Security Misconfiguration:

  • Security Misconfiguration happens when the system is not properly secured, allowing attackers to access the data
  • Security Misconfiguration is dangerous because it allows unauthorized users to access the data
  • Causes:
    1. Default Credentials:
      • The system still uses default usernames and passwords, allowing attackers to access the data
    2. Unnecessary Features:
      • The system uses unnecessary features such as test files and Debugging tools, allowing attackers to access the data
    3. Weak Configuration:
      • Development, Testing, and Production environments are not properly configured, allowing attackers to access the data
    4. Update Required:
      • The system uses old programs and libraries, allowing attackers to access the data
  • Prevention:
    • Change the default credentials
    • Stop using the unnecessary features
    • Provide a strong configuration
    • use new programs and libraries

7)Insecure Deserialization:

  • Insecure Deserialization happens when the app accepts untrusted data, allowing attackers to access the data
  • Risks:
    1. RCE:
      • RCE stands for Remote Code Execution
      • An Attacker inserts the malicious serialized object into the Application.
      • When the Application tries to deserialized, it executes the harmful objects
    2. Denial of Service:
      • An Attacker inserts the malicious serialized object into the Application
      • When the Application tries to deserialized, it slows down the Application and denies the services
    3. Integrity Loss:
      • An Attacker inserts the malicious serialized object into the Application
      • Here, the attacker changes the values in serialized object
      • When the application tries to deserialized, the Application uses changed values, resulting in Integrity Loss
    4. Authentication Bypass:
      • An Attacker inserts the malicious serialized object into the Application
      • Here, the attacker changes the session or authentication data and performs unauthorized actions

9)Cross Site Request Forgery (CSRF):

  • CSRF stands for Cross Site Request Forgery
  • In CSRF, it tricks the users into performing unwanted actions on a website without their knowing
  • How does it work?
    • User logs in to the trusted website
    • The attackers trick the users into clicking on harmful links
    • The attacker sends the fake HTTP requests to the trusted website
    • The trusted website accepts the HTTP requests because it thinks that the request is coming from an authorized user
  • Risks:
    1. Fake Transfer:
      • The money will be transferred from the account without the user knowing
    2. Fake Modifications:
      • The account details are modified without the user knowing
      • Example:
        • changing usernames and passwords without the user's permission
    3. reputation damage:
      • Failing to validate the user's request can damage the organization reputation
